Passwordless Sudo Requirement

Infrastructure installer buttons use sudo -n. This prevents the web request from hanging on an invisible password prompt. For these buttons to work, the Linux user running this panel must be allowed to run the needed sudo commands without an interactive password.

For a development VM, the simplest setup is:

sudo visudo

Add this at the bottom, replacing youruser with the Linux username running the web-admin:

youruser ALL=(ALL) NOPASSWD: ALL

Then verify:

sudo -l
sudo -n true
sudo -n apt update
Production hardening note: NOPASSWD: ALL is broad. For public/shared environments, replace it with a restricted command allowlist.
